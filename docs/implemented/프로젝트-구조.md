# 프로젝트 구조

## 개요

NestJS 기반 모노레포 프로젝트. User App과 Admin App 2개의 애플리케이션, common과 prisma 2개의 공유 라이브러리로 구성.

## 기술 스택

| 분류 | 기술 |
|------|------|
| 프레임워크 | NestJS 11, TypeScript 5.7 |
| 데이터베이스 | PostgreSQL + Prisma 7.2 |
| 캐시 | Redis (ioredis) |
| 인증 | Passport.js + JWT |
| 로깅 | Winston |
| 유효성 검증 | class-validator, class-transformer |
| API 문서 | Swagger (OpenAPI) |
| 헬스체크 | @nestjs/terminus |
| 패키지 관리 | pnpm workspace |

## 디렉토리 구조

```
nestjs-monorepo-sample/
├── apps/
│   ├── user/src/
│   │   ├── auth/              # 사용자 인증
│   │   ├── user/              # 사용자 프로필 관리 (UserMeController)
│   │   ├── post/              # 게시글 CRUD
│   │   ├── notification/      # 알림 수신/읽기
│   │   ├── config/            # 환경 설정
│   │   ├── app.module.ts
│   │   └── main.ts
│   └── admin/src/
│       ├── auth/              # 관리자 인증 (loginId 기반)
│       ├── user/              # 사용자 관리 (조회, 상태 변경, 삭제)
│       ├── post/              # 게시글 관리 (상태 변경, 삭제)
│       ├── notification/      # 알림 발송/목록 조회
│       ├── configs/           # 환경 설정
│       ├── app.module.ts
│       └── main.ts
├── libs/
│   ├── common/src/            # 공통 유틸리티 (가드, 전략, 서비스, 데코레이터 등)
│   └── prisma/src/            # DB 스키마, 시드, PrismaService
├── envs/                      # 공통 환경 변수
└── nest-cli.json
```

## 앱별 설정

| 항목 | User App | Admin App |
|------|----------|-----------|
| 기본 포트 | 3000 | 3001 |
| API 접두사 | `/api/v1` | `/admin/api/v1` |
| Swagger | `/api/docs` | `/admin/api/docs` |
| 인증 방식 | 이메일 로그인 | loginId 로그인 |

## 환경 변수 로딩

```
1단계: ./envs/.env.{NODE_ENV}               → 공통 (DB, Redis, JWT 등)
2단계: ./apps/{app}/envs/.env.{NODE_ENV}     → 앱 전용 (포트, 앱 이름)
```

## 모듈 의존성

```
User AppModule
├── CommonModule (@Global: CryptoService, TokenService, Strategies, Throttler, Redis)
├── PrismaModule (@Global)
├── AuthModule → UserModule
├── UserModule → PostModule, NotificationModule (UserMeController)
├── PostModule, NotificationModule
└── HealthModule

Admin AppModule
├── RouterModule (path: 'admin')
├── CommonModule, PrismaModule
├── AdminAuthModule (독립된 Admin 인증)
├── AdminUserModule, AdminPostModule, NotificationModule
└── HealthModule
```

## 부트스트랩 (main.ts)

두 앱 공통:
- GlobalExceptionHandler, ClassSerializerInterceptor, ValidationPipe 등록
- URI 기반 API 버전 관리 (`defaultVersion: '1'`)
- CookieParser, ShutdownHooks, Swagger 설정
- JwtAccessGuard를 `APP_GUARD`로 전역 등록
- LoggerMiddleware, CustomClsMiddleware 적용

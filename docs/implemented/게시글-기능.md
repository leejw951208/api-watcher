# 게시글 기능

## 개요

User App에서 게시글 CRUD, Admin App에서 게시글 관리(상태 변경, 삭제). Offset/Cursor 두 가지 페이지네이션, 조회수 추적, Soft Delete 지원.

## User App 기능

### 게시글 생성
- `status` 미지정 시 기본값 `PUBLISHED`
- `createdBy`는 CLS에서 자동 주입
- `@Permission('post', 'write')` 권한 필요

### 게시글 조회
- 본인의 게시글만 조회 가능 (`findByIdAndUserId`에 userId 조건)
- 조회 시 `viewCount` 자동 증가 (Prisma `{ increment: 1 }`)

### 게시글 수정/삭제
- 수정 가능 필드: `title`, `content`, `status` (선택적)
- 삭제: `isDeleted: true` + `deletedAt`, `deletedBy` 기록
- 둘 다 `@Permission('post', 'write/delete')` 권한 필요

### 페이지네이션

**Offset** (`GET /posts/offset`, `GET /posts/me/offset`):
- 쿼리: `page`, `size`, `order`, `title?`, `status?`
- `findMany`와 `count`를 병렬 실행
- 응답: `{ data: [...], meta: { page, totalCount } }`

**Cursor** (`GET /posts/cursor`, `GET /posts/me/cursor`):
- 쿼리: `lastCursor?`, `size`, `order`, `title?`
- `size + 1`개를 조회하여 다음 페이지 존재 여부 확인
- 응답: `{ data: [...], meta: { nextCursor } }`

| 비교 | Offset | Cursor |
|------|--------|--------|
| 성능 | 페이지 깊어질수록 느림 | 일정 |
| 데이터 일관성 | 삽입/삭제 시 중복 가능 | 보장 |
| UI | 페이지 번호 네비게이션 | 무한 스크롤 |

## Admin App 기능

관리자가 모든 사용자의 게시글을 관리. RBAC 권한 적용.

| 엔드포인트 | 설명 | Permission |
|-----------|------|------------|
| `GET /posts` | 전체 게시글 목록 (Offset) | `post:read` |
| `GET /posts/:id` | 게시글 상세 | `post:read` |
| `PATCH /posts/:id/status` | 상태 변경 (PUBLISHED, HIDDEN, DRAFT) | `post:update` |
| `DELETE /posts/:id` | Soft Delete | `post:delete` |

검색 조건: `title?`, `userId?`, `status?`

## 게시글 상태

| 상태 | 설명 |
|------|------|
| `DRAFT` | 초안 (비공개) |
| `PUBLISHED` | 발행됨 (공개) |
| `HIDDEN` | 숨김 처리 |

## DB 인덱스

```
@@index([userId, status, isDeleted])    # 내 게시글 조회
@@index([status, isDeleted, id])        # 전체 목록 + 정렬
```

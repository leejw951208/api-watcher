# 설계 패턴

## 레이어 아키텍처

```
Controller   → 라우트, 데코레이터, 응답 래핑, 쿠키 관리
Service      → 비즈니스 로직, 에러 판단, DTO 변환
Repository   → Prisma 쿼리, 감사 필드 입력, 페이지네이션
```

## Soft Delete

모든 모델에 적용. 물리 삭제 대신 `isDeleted: true` 마킹.

```typescript
// 삭제
data: { isDeleted: true, deletedAt: new Date(), deletedBy: this.cls.get('id') }

// 조회 — 모든 쿼리에 포함
where: { id, isDeleted: false }
```

## 감사 필드 (Audit Trail)

CLS 미들웨어에서 JWT를 디코딩하여 사용자 ID를 저장. Repository에서 자동 주입.

```typescript
// 생성
data: { ...data, createdBy: this.cls.get('id') }

// 수정
data: { ...data, updatedAt: new Date(), updatedBy: this.cls.get('id') }
```

## DTO 변환

```typescript
plainToInstance(ResponseDto, entity, { excludeExtraneousValues: true })
```

`@Expose()` 데코레이터가 붙은 필드만 응답에 포함. password, isDeleted 등 자동 제외.

## 병렬 처리

독립적인 작업은 `Promise.all`로 병렬 실행:

```typescript
// 토큰 생성
const [accessToken, refreshToken] = await Promise.all([...])

// 페이지네이션
const [items, totalCount] = await Promise.all([findMany(...), count(...)])

// 토큰 삭제 (부분 실패 허용)
await Promise.allSettled([redis.del(...), db.delete(...)])
```

## API 버전 관리

URI 기반 버전 관리 (`/api/v1/...`). `v1/` 폴더 분리 대신 파일명 기반.

```typescript
// main.ts
app.enableVersioning({ type: VersioningType.URI, prefix: 'v', defaultVersion: '1' })

// controller
@Controller({ path: 'posts', version: '1' })
```

v2 추가 시: 같은 모듈에 `post-v2.controller.ts`를 추가하고 `version: '2'` 지정.

## 앱 간 독립성

User App과 Admin App은 서로의 코드를 직접 참조하지 않음.
- 공유 인프라: `libs/common` (가드, 전략, 서비스)
- 인증 로직: 각 앱에 독립적인 AuthModule (User: email 기반, Admin: loginId 기반)

## 보안 패턴

| 패턴 | 구현 |
|------|------|
| Refresh Token Rotation | 사용 후 삭제 + 새 토큰 발급 |
| HTTP-Only 쿠키 | Refresh Token JS 접근 불가 |
| bcrypt 해싱 | 비밀번호, Refresh Token |
| timingSafeEqual | OTP 코드 비교 (타이밍 공격 방지) |
| 랜덤 지연 | 미존재 사용자 응답 (존재 여부 비노출) |
| 전체 세션 무효화 | 비밀번호 변경 시 모든 Refresh Token 삭제 |
| Redis Rate Limiting | INCR 원자적 카운팅 |
| RBAC | scope:action 기반 권한 검사 |
| ValidationPipe | whitelist + forbidNonWhitelisted |

# 공통 라이브러리 (`@libs/common`)

`@Global()` 모듈로 등록되어 모든 모듈에서 사용 가능.

## CommonModule 구성

```typescript
@Global()
@Module({
    imports: [CustomThrottlerModule, RedisModule.forRootAsync()],
    providers: [CryptoService, TokenService, JwtAccessStrategy, JwtRefreshStrategy],
    exports: [CryptoService, TokenService, JwtAccessStrategy, JwtRefreshStrategy]
})
```

## 미들웨어

### LoggerMiddleware
- 모든 요청에 대해 `{METHOD} {URL} {STATUS} {IP} {USER_AGENT}` 형식으로 로깅
- 응답 완료 시점(`res.on('finish')`)에 기록

### CustomClsMiddleware
- CLS에 요청 컨텍스트 저장: `id` (사용자 PK), `aud` (admin/user), `agent`, `clientIp`
- JWT 디코딩으로 추출 (검증 없이 파싱만)
- Repository에서 `cls.get('id')`로 감사 필드 자동 입력에 활용

## 데코레이터

| 데코레이터 | 용도 |
|-----------|------|
| `@Public()` | JwtAccessGuard 인증 우회 |
| `@CurrentUser()` | JWT 페이로드를 파라미터로 주입 |
| `@Permission(scope, action)` | RBAC 권한 메타데이터 설정 |
| `@ApiAuthGuard()` | Swagger 인증 정보 + 공통 에러 응답 자동 추가 |
| `@ApiExceptionResponse(errorCode)` | Swagger 에러 응답 문서화 |
| `@ApiOkBaseResponse(dto)` | Swagger 200 응답 문서화 |
| `@ApiOkOffsetPaginationResponse(dto)` | Swagger Offset 페이지네이션 응답 |
| `@ApiOkCursorPaginationResponse(dto)` | Swagger Cursor 페이지네이션 응답 |

## 예외 처리

### GlobalExceptionHandler

| 순서 | 예외 타입 | 처리 |
|------|----------|------|
| 1 | `BaseException` (커스텀) | 에러 코드/메시지 반환, logLevel에 따라 warn/error |
| 2 | `BadRequestException` (class-validator) | 유효성 검증 메시지 결합 |
| 3 | `NotFoundException` | NOT_FOUND_001 |
| 4 | 기타 | SERVER_ERROR_001 (500) |

## 공통 DTO

```typescript
ResponseDto<T>           // { data: T }
CreateResponseDto        // { data: number }
OffsetRequestDto         // page, size, order
CursorRequestDto         // lastCursor?, size, order
OffsetResponseDto<T>     // { data: T[], meta: { page, totalCount } }

```

## Rate Limiting

Redis 기반 분산 Rate Limiting. `INCR` 명령으로 원자적 카운팅.

```typescript
@UseGuards(CustomThrottlerGuard)
@Throttle({ ip: { limit: 5, ttl: 60000 }, user: { limit: 5, ttl: 60000 } })
```

- **추적 키**: 인증 사용자 → `user:{userId}`, 미인증 → `ip:{clientIp}`
- **Redis 키**: `throttler:{name}:{tracker}` (예: `throttler:user:user:123`)
- **기본 전역 설정**: IP/User 각 60초당 10,000회 (실질적으로 무제한, 엔드포인트별 오버라이드)

## Redis 모듈

ioredis 기반. `REDIS_CLIENT` 토큰으로 전역 주입.

- 재연결 전략: 최대 10회, 지연 `Math.min(times * 1000, 3000)ms`
- 연결/에러 이벤트 Winston 로깅

**Redis 키 패턴 전체:**

| 패턴 | 값 | TTL | 용도 |
|------|-----|-----|------|
| `rt:{owner}:{ownerId}:{jti}` | 해시된 Refresh Token | 7일 | 토큰 캐시 |
| `{owner}:{ownerId}:tokens` | JTI 배열 (JSON) | 7일 | 전체 삭제용 목록 |
| `password-reset:token:{token}` | userId | 5분 | 리셋 토큰 |
| `password-reset:rate:{email}` | 요청 횟수 | 5분 | 이메일 기준 Rate Limiting |
| `throttler:{name}:{tracker}` | 요청 횟수 | 설정 ttl | Rate Limiting |

## Health Check

`@nestjs/terminus` + `RedisHealthIndicator` (커스텀).

```
GET /api/v1/health          # User App
GET /admin/api/v1/health    # Admin App
```

- DB: `PrismaHealthIndicator.pingCheck()`
- Redis: `redis.ping()` → PONG 확인
- `@Public()` 데코레이터로 인증 불필요

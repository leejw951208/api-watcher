# 사용자 관리

## User App — 프로필 관리

인증된 사용자가 자신의 프로필을 조회/수정/탈퇴.

| 엔드포인트 | 설명 | 비고 |
|-----------|------|------|
| `GET /users/me` | 내 정보 조회 | Rate Limit: 5/분 (IP, User) |
| `PATCH /users/me` | 내 정보 수정 | 수정 가능: email, phone |
| `DELETE /users/me` | 회원 탈퇴 | Soft Delete |

- `plainToInstance`의 `excludeExtraneousValues: true`로 password 등 민감 필드 자동 제외
- 탈퇴 시 `status: WITHDRAWN`, `isDeleted: true` 설정
- 이미 탈퇴한 사용자 재탈퇴 방지 (`USER_ERROR.ALREADY_DELETED`)

## Admin App — 사용자 관리

관리자가 전체 사용자를 조회/상태 변경/삭제. 모든 엔드포인트에 RBAC 권한 적용.

| 엔드포인트 | 설명 | Permission |
|-----------|------|------------|
| `GET /users` | 사용자 목록 (Offset) | `user:read` |
| `GET /users/:id` | 사용자 상세 | `user:read` |
| `PATCH /users/:id/status` | 상태 변경 (ACTIVE, INACTIVE, WITHDRAWN) | `user:update` |
| `DELETE /users/:id` | Soft Delete | `user:delete` |

**검색 조건**: `name?`, `email?`, `status?`

## UserRepository 메서드

| 메서드 | 설명 | 사용처 |
|--------|------|--------|
| `findById(id)` | ID로 조회 (isDeleted: false) | User/Admin |
| `findByEmail(email)` | 이메일로 조회 | 로그인 |
| `findByEmailAndPhone(email, phone)` | 이메일+휴대폰으로 조회 | 비밀번호 재설정 |
| `existsByEmail(email)` | 이메일 존재 여부 | 회원가입 |
| `create(data)` | 사용자 생성 | 회원가입 |
| `update(id, data)` | 정보 수정 | 프로필 수정 |
| `updatePassword(id, hash)` | 비밀번호 변경 | 비밀번호 재설정 |
| `softDelete(id)` | Soft Delete (WITHDRAWN) | 탈퇴 |

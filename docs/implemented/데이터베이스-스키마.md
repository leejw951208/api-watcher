# 데이터베이스 스키마

## 설정

- **DB**: PostgreSQL, 멀티 스키마 (`public`, `base`, `main`)
- **ORM**: Prisma 7.2 + `@prisma/adapter-pg` (커넥션 풀 max 10)
- **스키마 분리**: `libs/prisma/src/configs/models/` 디렉토리에 모델별 `.prisma` 파일

## 공통 필드

모든 모델에 적용:

```
createdAt  DateTime   @default(now())   # 생성 시각
createdBy  Int?                          # 생성자 ID (CLS에서 자동 주입)
updatedAt  DateTime?                     # 수정 시각
updatedBy  Int?                          # 수정자 ID
isDeleted  Boolean?   @default(false)    # Soft Delete 여부
deletedAt  DateTime?                     # 삭제 시각
deletedBy  Int?                          # 삭제자 ID
```

## Enum

| Enum | 값 |
|------|-----|
| `UserStatus` | ACTIVE, INACTIVE, WITHDRAWN |
| `Owner` | ADMIN, USER |
| `TokenType` | JWT |
| `PostStatus` | DRAFT, PUBLISHED, HIDDEN |
| `NotificationType` | SYSTEM, POST, COMMENT, USER |

## 모델

### User (`base.user`)
- `id`, `email` (unique), `password`, `name`, `phone`, `status` (default: ACTIVE)
- 관계: Post[], UserRole[], Notification[], NotificationRead[]

### Admin (`base.admin`)
- `id`, `loginId` (unique), `password`, `email` (unique), `name`, `phone`, `status`
- 관계: AdminRole[]

### Post (`base.post`)
- `id`, `title` (varchar 200), `content` (text), `userId` (FK→User), `viewCount`, `status` (default: PUBLISHED)
- 인덱스: `[userId, status, isDeleted]`, `[status, isDeleted, id]`

### Token / TokenJwt (`base`)
- Token: `tokenHash`, `tokenType` (JWT), `owner` (ADMIN|USER), `ownerId`
- TokenJwt: `tokenId` (FK→Token), `jti`

### Notification / NotificationRead (`public`)
- Notification: `userId?` (null=전체 공지), `title` (varchar 100), `content` (varchar 500), `type`
- NotificationRead: `userId` (FK), `notificationId` (FK) — 읽음 기록 별도 관리
- 인덱스: `[userId, createdAt]`

### Role / Permission / RolePermission (`base`)
- Role: `name`, `owner` (ADMIN|USER), `description?`
- Permission: `scope`, `action` — unique `[scope, action]`
- RolePermission: `roleId`, `permissionId` — unique `[roleId, permissionId]`

### UserRole / AdminRole (`base`)
- UserRole: `userId`, `roleId` — unique `[userId, roleId]`
- AdminRole: `adminId`, `roleId` — unique `[adminId, roleId]`

## ER 다이어그램

```
User ──1:N──> Post
User ──1:N──> UserRole ──N:1──> Role ──1:N──> RolePermission ──N:1──> Permission
User ──1:N──> NotificationRead ──N:1──> Notification
Admin ──1:N──> AdminRole ──N:1──> Role
Token ──1:N──> TokenJwt
```

# 인증 및 인가

## 토큰 구조

```typescript
type JwtPayload = {
    id: number          // 사용자/관리자 PK
    type: 'ac' | 're'   // Access / Refresh
    aud: 'admin' | 'user'
    jti: string          // UUID
    issuer: 'monorepo'
}
```

| 토큰 | 기본 TTL | 저장 위치 |
|------|----------|-----------|
| Access Token | 1시간 | 응답 Body |
| Refresh Token | 7일 | HTTP-Only 쿠키 (웹) / 헤더 (앱) |

## User 인증 흐름

### 회원가입 → 로그인 → 로그아웃

- **회원가입**: 이메일 중복 검사 → 비밀번호 bcrypt 해싱 → 사용자 생성
- **로그인**: 이메일로 조회 → 비밀번호 비교 → JTI 생성 → Access/Refresh 토큰 병렬 발급 → Refresh Token을 Redis + DB 이중 저장
- **로그아웃**: Refresh Token 검증 → Redis + DB에서 해당 토큰 삭제

### 토큰 재발급 (Refresh Token Rotation)

1. 기존 Refresh Token 검증 + Redis 캐시 비교
2. 기존 토큰 삭제 (일회용)
3. 새 JTI로 새 토큰 쌍 발급 + 저장

### 비밀번호 재설정 (2단계)

1. **본인 확인** (`POST /auth/password-reset/request`): 이메일 + 휴대폰번호 DB 검증 → resetToken 발급 (5분 TTL)
2. **비밀번호 변경** (`PATCH /auth/password-reset`): resetToken 검증 → 비밀번호 변경 → 모든 Refresh Token 삭제

**Rate Limiting (이중 제한):**
- IP 기반: `@Throttle` + `CustomThrottlerGuard` — 5회/5분
- 이메일 기반: Redis `INCR` 카운팅 — 5회/5분 (`password-reset:rate:{email}`)

**보안 고려사항**: 이메일+휴대폰 이중 검증, 이중 Rate Limiting, 비밀번호 변경 시 전체 세션 무효화

## Admin 인증

User와 별도의 `AdminAuthModule`로 분리. Admin 모델의 `loginId` 기반 로그인.

- **로그인** (`POST /admin/api/v1/auth/signin`): loginId + password → Access/Refresh 토큰 발급
- **로그아웃** (`DELETE /admin/api/v1/auth/signout`)
- **토큰 재발급** (`POST /admin/api/v1/auth/token/refresh`)

인증 인프라(가드, 전략, TokenService, CryptoService)는 `libs/common`을 공유하고, 비즈니스 로직만 앱별로 분리.

## 가드

| 가드 | 역할 | 적용 |
|------|------|------|
| `JwtAccessGuard` | Access Token 검증, `@Public()` 시 우회 | `APP_GUARD` 전역 |
| `JwtRefreshGuard` | Refresh Token 검증 (헤더 or 쿠키) | 토큰 재발급 엔드포인트 |
| `PermissionGuard` | RBAC scope:action 권한 검사 | `@Permission()` + `@UseGuards()` |
| `CustomThrottlerGuard` | IP/사용자 기반 Rate Limiting | `@Throttle()` + `@UseGuards()` |

## RBAC (역할 기반 접근 제어)

```
User/Admin → UserRole/AdminRole → Role → RolePermission → Permission(scope, action)
```

**적용된 엔드포인트:**

| 앱 | 엔드포인트 | Permission |
|----|-----------|------------|
| User | `POST /posts` | `post:write` |
| User | `PATCH /posts/:id` | `post:write` |
| User | `DELETE /posts/:id` | `post:delete` |
| Admin | `POST /notifications` | `notification:write` |
| Admin | `GET /notifications` | `notification:read` |
| Admin | `GET/PATCH/DELETE /users/*` | `user:read/update/delete` |
| Admin | `GET/PATCH/DELETE /posts/*` | `post:read/update/delete` |

**시드 역할:**
- `SUPER_ADMIN`: 모든 권한
- `ADMIN`: user, post, notification 관련 권한
- `USER`: post 관련 권한

## 토큰 저장소

**Redis (캐시):**
```
rt:{owner}:{ownerId}:{jti}     → 해시된 Refresh Token (TTL: 7일)
{owner}:{ownerId}:tokens       → JTI 목록 JSON 배열 (TTL: 7일)
```

**DB (영속):**
```
Token { tokenHash, tokenType(JWT), owner, ownerId }
TokenJwt { tokenId(FK), jti }
```

삭제 시 `Promise.allSettled`로 Redis/DB 부분 실패를 허용.
